Alleviate

A responsive website for a multispeciality hospital

Overview

About the project

My residence in the Indian city of Hyderabad was situated right next to a major multispeciality hospital. Disappointingly however, the hospital lacked a proper website. As a personal project, I decided to build a website for a multispeciality website that can be adopted my any hospital.

The website has been developed from scratch using HTML, CSS and JavaScript. Prior to the development, I did a market analysis and competative analysis, did user research and some low-fidelity designs.

Tasks:
User Research, Competative analysis, Low fidelity design, Front-end development

Duration:
two weeks, Dec' 18 to Jan'19

The Challenge

Developing a responsive website that provides a smooth and immersive experience to the user. The website should be easy to navigate, and should be able to provide all the essential information to the users.

The Solution

Alleviate Web Application

A responsive web application that provides details about a multispeciality hospital to the user. The website is easy to navigate and intuititve to use.

Conclusion

Learning

I honed my front-end development skills through building this website. This has been one of the prelimiary projects I did in front-end development, and this was instrumental for me in further enhancing my interest and passion for front end development and software engineering. I also learnt to think from the perspectives for the end users and WYSIWYG design.

This tool has a lot of potential for being adpoted by hospitals.This serves as a template that hospitals can modify according to their need. I have kept the code for this website open to public access on github and have sent several multispeciality hospitals in Hyderabad area access to it in case they require it.
